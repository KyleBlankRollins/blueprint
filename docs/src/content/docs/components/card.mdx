The `bp-card` component provides a styled container with optional header, footer, and media slots.

## Import

```javascript
import 'blueprint/components/card';
```

### Horizontal Layout

Use `direction="horizontal"` to place media beside the body content. Header and footer remain full-width.

<ComponentPreview>
  <bp-card direction="horizontal" variant="elevated">
    <img slot="media" src="https://placecats.com/250/200" alt="A cat" style="width: 250px; height: 200px; object-fit: cover;" />
    <bp-heading level="4" size="md">Horizontal Card</bp-heading>
    <bp-text>Content sits next to the image when the direction is set to horizontal. This is useful for article previews, product listings, or any layout that benefits from a side-by-side arrangement.</bp-text>
  </bp-card>
</ComponentPreview>

```html
<bp-card direction="horizontal" variant="elevated">
  <img slot="media" src="https://placecats.com/250/200" alt="A cat" style="width: 250px; object-fit: cover;" />
  <bp-heading level="4" size="md">Horizontal Card</bp-heading>
  <bp-text>Content sits next to the image.</bp-text>
</bp-card>
```

## API Reference

### Properties

| Property    | Type                                    | Default      | Description                              |
| ----------- | --------------------------------------- | ------------ | ---------------------------------------- |
| `variant`   | `'default' \| 'outlined' \| 'elevated'` | `'default'`  | Visual style variant                     |
| `hoverable` | `boolean`                               | `false`      | Adds hover effect                        |
| `clickable` | `boolean`                               | `false`      | Makes card interactive                   |
| `noPadding` | `boolean`                               | `false`      | Removes default padding                  |
| `direction` | `'vertical' \| 'horizontal'`            | `'vertical'` | Layout direction for the content area    |

### Slots

| Slot      | Description                          |
| --------- | ------------------------------------ |
| (default) | Main card content                    |
| `header`  | Card header section                  |
| `footer`  | Card footer section                  |
| `media`   | Media content (images, videos, etc.) |

### Events

| Event   | Detail | Description                       |
| ------- | ------ | --------------------------------- |
| `click` | -      | Fired when clickable card clicked |

### CSS Parts

| Part      | Description                            |
| --------- | -------------------------------------- |
| `card`    | The card container                     |
| `header`  | The header section                     |
| `content` | The content area wrapping media + body |
| `body`    | The body content area                  |
| `media`   | The media slot container               |
| `footer`  | The footer section                     |

## Accessibility

- Clickable cards use `role="button"` and are keyboard accessible
- Focus styles are clearly visible
- Interactive states are properly announced
